12. INCEpTION¬∂

12.1. The big picture¬∂

In order to train a model, you need annotated text. The algorithm that produces the model needs to see lots of examples of all the different kinds of annotation that you want it to be able to create for new texts. For some kinds of annotations, like part-of-speech tagging, you‚Äôll definitely encounter lots of examples of things like nouns, verbs, and determiners in almost any text you choose. But if you want the model to be able to recognize less-common verb tenses, or classifier particles that only occur with a set of relatively rare words, you‚Äôll need to be deliberate about choosing the texts you annotate in order to include multiple examples of those things, too. Similarly if you‚Äôre trying to annotate your texts so that your model can do named entity recognition: you may have seen models that can identify people‚Äôs names vs. place names vs. organization names with some amount of accuracy. It‚Äôs a lot easier to get a model to recognize that a word is some kind of entity; to get any accuracy in differentiating entity types, like person vs. place names, you need lots of examples of how and where both kinds of entity are used.

Realistically, you‚Äôll end up making your best guess about what text to annotate. You‚Äôll annotate them, train the model, and then annotate some more text based on where the model isn‚Äôt doing the right thing. And then repeat the process over again.

12.2. This tutorial¬∂

This video/tutorial will walk you through how to annotate a text using Inception, a free, open-source, collaborative annotation environment.

We‚Äôll cover what your text will look like when you get it out of Cadet, how to load it into Inception, how to do some of the most common types of annotation, and how to export your annotations when you‚Äôre done.

While this video/tutorial will cover the things we‚Äôve thought of, there may be particular things you need to annotate that we won‚Äôt cover here. If you have questions or feel stuck, please reach out to the instructors! We‚Äôll be revising these materials based on your feedback.

At a minimum, Cadet will create a spaCy language object using your data, and will automatically split your text into sentences and words. What you‚Äôll have gotten out of Cadet is a file with a .conllu extension, which is a plain-text file with a special way of presenting the text. Each word (or other token, which means things like punctuation) will be on its own line, there‚Äôll be a number of dashes, and each sentence will be separated by a space.

12.4. Welcome to Inception¬∂

The first step is to log into the Inception instance set up for the institute.

Once you‚Äôve logged in, you‚Äôll go to the Projects page, where you can see all the projects you have access to and create new ones.

Click on the blue ‚ÄúCreate new project‚Äù button.

12.5. Creating a new project¬∂

Give your project a name, and write a brief description if you‚Äôd like.

If your language uses a right-to-left script, use the drop-down for script direction to indicate that.

Hit the save button to move on to the next page of settings. These are the same options you‚Äôll see when you go into the ‚ÄúSettings‚Äù area of your project‚Äôs configuration page later, so you can easily change things if needed.

12.5.1. Documents¬∂

Choose the ‚ÄúDocuments‚Äù tab. Click the file-folder icon to browse your computer for the .conll file that you downloaded from Inception, and select ‚ÄúCoNLL-U‚Äù as the file format. Then click ‚ÄúImport‚Äù.

You should see the file appear in the list of files in the ‚ÄúDocuments‚Äù tab.

12.5.2. Users¬∂

Under the ‚ÄúUsers‚Äù tab, you can add the annotators who will be working on your project with you. If you want to add an existing user to your project (e.g. Institute instructors), you can choose them from the dropdown. If you need to create a new user account, contact an Institute instructor; you need superuser permissions to create a new user.

By default, any user you add just has the role of ‚Äúannotator‚Äù. If you click on their name, you can add additional roles like ‚ÄúManager‚Äù (manages the whole project) or ‚ÄúCurator‚Äù (can compare and reconcile different people‚Äôs annotations).

12.5.3. Layers¬∂

Layers are different kinds of annotation. When you‚Äôre annotating text, you have to specify what layer each annotation belongs to. You can create custom layers, but the built-in options should cover a lot of what you need.

All annotations are of the type span, relation, or chain.

12.5.3.1. Spans¬∂

Spans let you select one or more tokens next to each other. You use span annotations for part-of-speech tagging (you select each word and label it), named entity tagging (you may need to choose more than one word, like a person‚Äôs first and last names).

12.5.3.2. Relations¬∂

Relations allow you to connect tokens. The dependency layer uses relations, for instance, to connect text that‚Äôs been labeled with part-of-speech annotations. Things like connecting a verb with the subject performing that action.

12.5.3.3. Chains¬∂

Chains define relationships between multiple things at the same time. The most common example of a chain is coreference resolution, where you want to connect all the words and pronouns and names that refer to the same person.

12.5.3.4. Key bindings¬∂

Once you start doing a lot of annotation, you may want to set up keyboard shortcuts, or ‚Äúkey bindings‚Äù, so you don‚Äôt have to select annotation values using the drop-downs. For detailed documentation, including which key combinations are permissible, see the Inception documentation.

To set up a key binding, choose a layer, then choose a feature in the upper right corner. If you scroll down through the feature details, at the bottom right you‚Äôll see key bindings, where you can map a particular label to a particular key combination.

12.5.4. Tagsets¬∂

To make your model maximally compatible with other models, it‚Äôs ideal if you can use standard sets of tags (like Universal Dependency tags for part-of-speech and dependency relations). Depending on what you want to do with your model, you may need to add custom tagsets, e.g. formal vs. informal vs. nickname form of names. If you want to add new values to an existing tagset, click on the name of the tagset and then hit the blue ‚ÄúCreate‚Äù button above the list of tags. If you create an entirely new tagset, you‚Äôll need to make sure you connect it to one of your layers back in the Layers tab.

When you‚Äôre done setting up your new project, click the gray ‚ÄúClose‚Äù button in the upper right.

12.6. Projet dashboard¬∂

On the projects page you see when you‚Äôve logged into Inception, clicking on a project will take you to the project dashboard.

12.7. Annotation¬∂

When you click the ‚ÄúAnnotation‚Äù icon, as an administrator on your project, you‚Äôll get to choose which user you‚Äôre annotating the text as, and which text file you‚Äôre annotating.

If you already used Cadet to automatically annotate your text, you‚Äôll see those annotations when you load the annotation screen. The annotation screen shows five sentences at a time, by default.

If Cadet didn‚Äôt do any automatic annotation, you‚Äôll see the sentences are split into lines, and nothing more.

12.7.1. Layer selection¬∂

In the column on the right, choose a layer. For this example, we‚Äôll start with part-of-speech

12.7.2. Annotating single/multiple tokens¬∂

For layers that require you to work with a single token at a time (like part-of-speech), double-click a single token. The right column will display the layer you‚Äôre working with and text you‚Äôve selected. You can use the dropdown for UPOS (universal part-of-speech) to select the right part-of-speech tag. Once you‚Äôve selected the tag, it will appear above the token you‚Äôve selected. If you need to delete it, click on the tag above the text and hit the red ‚ÄúDelete‚Äù button in the right column.

Some layers, like named entity recognition, allow you to select multiple adjacent tokens. For those layers, you can highlight the texts that should be annotated.

12.7.3. Dependencies¬∂

The dependency layer isn‚Äôt selectable, but you can add dependency annotations from the part-of-speech layer. Click on one token, and drag it onto the other token you want to connect it with. The right sidebar will be populated with the source (‚Äúfrom‚Äù) and the target (‚Äúto‚Äù), and you can choose a label using the ‚ÄúRelation‚Äù dropdown. Buttons at the top allow you to reverse the ‚Äúfrom‚Äù and ‚Äúto‚Äù values.

12.7.4. Wrapping up annotation¬∂

When you‚Äôre done with annotating a text and ready for the next step, hit the lock icon in the toolbar above the sentences. You‚Äôll see a warning pop up; hit the ‚ÄúConfirm‚Äù button.

12.8. Agreement¬∂

Inception provides an easy way to compare how often annotators are doing the same thing. This can be useful to monitor as a way to check in on your annotators, and identify any issues related to how to mark things up that you should discuss in a team meeting.

On the page for your project, click ‚ÄúAgreement‚Äù, then choose a feature, like POS. Cohen‚Äôs Kappa works as a default measure.

You‚Äôll see how many ‚Äúpositions‚Äù (or tokens) pairs of annotators have both done, as well as the percent of agreement:

Here you can see that the user accounts qad and qad2 have annotated 21 of the same ‚Äúpositions‚Äù and agree 94% of the time.

12.9. Curation¬∂

In Inception, curation is the process of reconciling differences in the annotation, and pulling together annotation chunks from different annotators. Each sentence in a document is given a number in the left-hand column.

Click on a number to view the annotations, and choose which layer you want to look at from the column on the right.

The view on the top is Inception‚Äôs attempt to reconcile the annotations from the various annotators. If you click on an annotation from an individual annotator, it will add that annotation to the final version on top.

In this example, we see in blue a number of annotations that were not included in the final version, either because there was a conflict between annotators (e.g. PROPN vs PRON for ‚ÄúShe‚Äù) or because one annotator omitted them (e.g. the punctuation). Clicking on those labels will add them to the final version. If they were originally omitted because of a conflict between annotators, the annotation you selected will turn green, and the alternate/incorrect annotation will turn red.

12.10. Export¬∂

When your data is ready to export for model training, go back to the Settings interface, and choose ‚ÄúExport‚Äù. You can select a format (we recommend CoNLL-U), then click the ‚ÄúExport the whole project‚Äù button. A zip file will automatically download once it is ready.
